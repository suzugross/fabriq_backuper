============================================================
Fabriq BackUper - 資格情報バックアップ (使い方)
============================================================

このフォルダは、バックアップ元 PC の Windows 資格情報マネージャに
登録されていたターゲット (URL / サーバ名 / アプリ識別子等) と
ユーザ名の一覧です。**パスワードはバックアップに含まれません**
(Windows の DPAPI 仕様により別 PC では復号できないため)。

新しい PC でこの一覧をもとに資格情報を再登録するには、各エントリ
ごとに **その場でパスワードを再入力** していただく必要があります。


-------------------------------------------------------------
含まれるファイル
-------------------------------------------------------------

  【フォルダ先頭 (ダブルクリックで使うもの)】
  登録.bat                一覧の資格情報をまとめて登録します
                          (エントリごとにパスワードを対話入力)。
  資格情報を表示.bat       旧PCにどの資格情報が有ったかを一覧表示する
                          ビューア (確認用。登録はしません)。まずこちらで
                          内容を確認し、どれを登録するか判断できます。

  【_data\ フォルダ (補助ファイル。通常は開かなくて構いません)】
  credentials_list.csv      一覧データ (Target / UserName / Type /
                            Persist / RestoreHint 等)
  register_credentials.ps1  登録の本体スクリプト (登録.bat が起動)
  Show-Credentials.ps1      ビューアの本体スクリプト (資格情報を表示.bat が起動)
  README.txt                本ファイル


-------------------------------------------------------------
復元手順
-------------------------------------------------------------

1. **新しい PC に、復元対象のユーザでログインしている状態にしてください。**
   資格情報マネージャは現在ログイン中のユーザに紐付くため、対象
   ユーザでログインしていない場合は意図しないユーザの資格情報
   マネージャに登録されてしまいます。

   ※ 先に **「資格情報を表示.bat」** を開くと、旧PCに有った資格情報を
      一覧で確認できます (登録はされません)。

2. フォルダ先頭の **「登録.bat」** をダブルクリックしてください。
   黒い画面 (コンソール) が開き、PowerShell スクリプトが起動します。
   ※ 管理者権限は必要ありません (むしろ、管理者で実行すると別
      ユーザの資格情報マネージャに登録されてしまうため避けてください)。

3. CSV の各エントリが順番に表示されます。
   各エントリには以下が表示されます:
     - Target  (資格情報の識別子: URL / サーバ名 / アプリ独自の ID)
     - Type    (Generic / DomainPassword / DomainCertificate 等)
     - UserName
     - Persist (Session / LocalMachine / Enterprise)

4. 各エントリで以下のいずれかを行います:
     - **パスワードを入力 → Enter** : そのエントリで登録 (既に同 Target/
       Type が登録済みでも問答無用で上書き、確認は入りません)
     - **何も入力せず Enter (空入力)** : そのエントリをスキップ

   ※ 証明書ベース (Type=DomainCertificate / GenericCertificate) の
      エントリはパスワード再入力では復元できないため、自動的にスキップ
      されます (パスワード入力プロンプトは出ません)。


-------------------------------------------------------------
情報注記 ("blob 長 0 のトークン/参照系の可能性あり")
-------------------------------------------------------------

エントリの中にはパスワード本体を持たないトークン/参照系のものが
混じっています。これらは元 PC で BlobSize=0 として観測されていた
エントリで、登録自体は可能ですが、移行先で意図通り動作しない場合が
あります。

該当パターン:
  - **トークン系 Generic** (MicrosoftAccount:user= / WindowsLive: /
    OneDrive / DriveFS_ / Office16_Data: 等)
      OAuth/refresh トークン や SSO バインドが格納されているため、
      パスワード再入力では原状復元できません。各アプリ (Outlook /
      Edge / OneDrive 等) を **起動して再サインイン** することで OS が
      自動的に再生成します。**スキップ (空 Enter) 推奨**。

  - **SspiPfc** 系の Domain 資格情報 (Comment=SspiPfc, BlobSize=0)
      ログオンセッションに紐付く参照エントリで、独立した password を
      持っていません。SMB 共有等で必要であれば普通にパスワードを
      入力して構いません (通常の Domain 資格情報として登録されます)。

  - **証明書系** (DomainCertificate / GenericCertificate)
      上記の通り自動スキップ。証明書ストアから別途インポートが必要。

なお、Windows OS が自動再生成する完全ノイズ (例:
`MicrosoftAccount:target=SSO_POP_Device`,
`WindowsLive:target=virtualapp/didlogical`) は **バックアップ時点で
除外** されているため、本 CSV には含まれません (v0.19.2 以降)。


-------------------------------------------------------------
登録後の確認
-------------------------------------------------------------

登録に成功した資格情報は以下で確認できます:

  - コマンドプロンプトで **cmdkey /list** を実行
  - コントロール パネル → ユーザー アカウント → 資格情報マネージャ
  - 「Windows 資格情報」または「汎用資格情報」タブ


-------------------------------------------------------------
復元完了後のフォルダ取り扱い
-------------------------------------------------------------

復元完了後、このフォルダ全体は不要になります。CSV には Target
(URL / サーバ名等) と UserName のペアが平文で記載されていますので、
**ご自身の責任で削除する** ことをおすすめします。

(パスワードは含まれていませんので、ファイル単体での漏洩リスクは
 低いですが、念のため。)

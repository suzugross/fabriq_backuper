============================================================
Fabriq BackUper - 資格情報バックアップ (使い方)
============================================================

このフォルダは、バックアップ元 PC の Windows 資格情報マネージャに
登録されていたターゲット (URL / サーバ名 / アプリ識別子等) と
ユーザ名の一覧です。**パスワードはバックアップに含まれません**
(Windows の DPAPI 仕様により別 PC では復号できないため)。

新しい PC でこの一覧をもとに資格情報を再登録するには、各エントリ
ごとに **その場でパスワードを再入力** していただく必要があります。


-------------------------------------------------------------
含まれるファイル
-------------------------------------------------------------

  credentials_list.csv    一覧データ (Target / UserName / Type /
                          Persist / RestoreHint 等を列挙)
  登録.bat                クリック起動口 (register_credentials.ps1
                          を PowerShell で起動するラッパー)
  register_credentials.ps1
                          本体スクリプト。CSV を読み、エントリごとに
                          パスワードを対話入力させて Windows 資格情報
                          マネージャに登録します。
  README.txt              本ファイル


-------------------------------------------------------------
復元手順
-------------------------------------------------------------

1. **新しい PC に、復元対象のユーザでログインしている状態にしてください。**
   資格情報マネージャは現在ログイン中のユーザに紐付くため、対象
   ユーザでログインしていない場合は意図しないユーザの資格情報
   マネージャに登録されてしまいます。

2. このフォルダ内の **「登録.bat」** をダブルクリックしてください。
   黒い画面 (コンソール) が開き、PowerShell スクリプトが起動します。
   ※ 管理者権限は必要ありません (むしろ、管理者で実行すると別
      ユーザの資格情報マネージャに登録されてしまうため避けてください)。

3. CSV の各エントリが順番に表示されます。
   各エントリには以下が表示されます:
     - Target  (資格情報の識別子: URL / サーバ名 / アプリ独自の ID)
     - Type    (Generic / DomainPassword / DomainCertificate 等)
     - UserName
     - Persist (Session / LocalMachine / Enterprise)
     - Hint    (cred-write = 再入力で復元可 / manual = 不可)

4. エントリごとにパスワードを入力 → Enter で次のエントリへ。
   - パスワードを入力せず Enter (空入力) でそのエントリをスキップ
   - 既に同じ Target/Type が登録されている場合は上書きの確認が出ます


-------------------------------------------------------------
スキップされるエントリ (Hint=manual)
-------------------------------------------------------------

以下のエントリは「パスワード再入力では復元できない」と判定され、
デフォルトでスキップされます (続行確認の y/N プロンプトが出ます):

  - **トークン系 Generic** (MicrosoftAccount: / WindowsLive: /
    OneDrive / DriveFS_ / Office16_Data: 等)
      OAuth/refresh トークンや SSO バインドが格納されているため、
      パスワードを入れても復元できません。各アプリ (Outlook / Edge /
      OneDrive 等) を起動して再サインインすることで自動的に再登録
      されます。

  - **証明書系** (DomainCertificate / GenericCertificate)
      証明書ストアから別途証明書をインポートする必要があります。

  - **blob 長 0** のエントリ
      パスワード本体を保持していない参照エントリ (アカウントが
      別の場所で管理されている等)。再入力では復元できません。


-------------------------------------------------------------
登録後の確認
-------------------------------------------------------------

登録に成功した資格情報は以下で確認できます:

  - コマンドプロンプトで **cmdkey /list** を実行
  - コントロール パネル → ユーザー アカウント → 資格情報マネージャ
  - 「Windows 資格情報」または「汎用資格情報」タブ


-------------------------------------------------------------
復元完了後のフォルダ取り扱い
-------------------------------------------------------------

復元完了後、このフォルダ全体は不要になります。CSV には Target
(URL / サーバ名等) と UserName のペアが平文で記載されていますので、
**ご自身の責任で削除する** ことをおすすめします。

(パスワードは含まれていませんので、ファイル単体での漏洩リスクは
 低いですが、念のため。)
